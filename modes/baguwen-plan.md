# Mode 6: 八股文背诵建议 (baguwen-plan)

## 目标

基于用户画像中的八股文 mastery 角色分类和目标公司，搜索真实面经题目，生成个性化学习计划。

## 前置条件

- `data/profile.yml` 应已存在，`baguwen_mastery.categories` 已根据角色激活
- `references/baguwen-role-taxonomy.md` 提供角色分类体系
- `references/company-priority-map.md` 提供公司-八股文考点映射

## 执行流程

### Step 1: 加载画像

读取 `data/profile.yml`：
- `baguwen_mastery.role`（当前角色）
- `baguwen_mastery.categories`（激活的分类）
- `target_roles.companies`（目标公司）
- `preferences.weak_areas_priority`（已知薄弱项）

### Step 2: 公司-分类优先级计算

从 `references/company-priority-map.md` 读取目标公司的八股文侧重。

```
对每个激活的分类：
  priority_score = 当前 target_roles.companies 对该分类的总体重视程度
  gap = priority_score × (6 - current_level)
```

### Step 3: WebSearch 搜索真实面经

对 gap 最高的 3-4 个分类，搜索真实面经：

搜索策略（每个分类搜 2-3 次，覆盖不同平台）：
```
site:nowcoder.com "[公司名] [分类关键词] 面经"
site:xiaohongshu.com "[公司名] [分类] 面试"
site:zhihu.com "[公司名] [分类] 面试题"
site:maimai.cn "[公司名] [技术关键词]"
```

从搜索结果中提取：
- 真实被问到的题目
- 来源平台和时间（标注如"牛客 2025"）
- 关联公司
- 题目出现频率（多次出现 > 单次提及）
- 难度（基础/进阶/深入）

**搜索优先级：**
1. 先搜精确公司名（"字节跳动 MySQL 索引 面经"）
2. 再搜分类通用面经（"Java 后端 MySQL 高频面试题"）
3. 如果某个平台返回空，换平台

### Step 4: Gap 分析

对每个激活的分类：
- P0（最高优先）：level ≤ 2 且目标公司高频 → 本周重点
- P1（高优先）：level ≤ 2 但不是目标公司最重点 → 两周内
- P2（中优先）：level = 3 → 巩固复习
- P3（低优先）：level ≥ 4 → 保持即可

### Step 5: 生成 2 周学习计划

```
## 📚 八股文学习计划

**目标岗位：** [角色名称]
**目标公司：** [公司列表]
**当前八股文水平：** [X]/5 | 已复习 [N] 题

---

### 第 1 周

#### 周一：[分类1]（P0 | [公司名]高频）

**[子主题1]**
1. "[真题]" — 🏷️ [公司名]面经 | 📎 牛客 2025
   - 考察点：[核心知识点]
   - 参考要点：[简要列出]

2. "[真题]"
   - ...

**[子主题2]**
...

#### 周二：[分类2]
...

#### 周三-周四：[分类3]（薄弱项）
...

#### 周五-周六：复习本周内容 + 自测
- 对照题目，不看答案自己讲一遍
- 标记讲得不流畅的，第二天重讲

#### 周日：休息 / 补漏

### 第 2 周
...

---

### 学习技巧

1. **不要死记硬背** —— 理解原理，用自己的话讲出来
2. **每天 30-45 分钟** —— 碎片时间够用，关键是坚持
3. **讲给镜子听** —— 能讲清楚才算真正掌握
4. **录音回听** —— 发现自己说了多少"然后...然后..."
5. **用 `/jobsfind-cn 八股文练习`** 开始交互式练习
```

### Step 6: 保存并引导

将计划写入 `data/baguwen-progress.md`。

> "八股文学习计划已生成。想开始练习吗？运行 `/jobsfind-cn 八股文练习`，我来考你一道。"

## 计划更新

- "八股文水平有变化" → 重算 Gap → 调整优先级
- "换目标公司" → 重搜面经 → 重排计划
- "特定分类想深入" → 该分类提升优先级

## 注意事项

- 搜索不到面经时明确告知用户，用"预测题"替代，标注「基于 JD 和行业经验预测」
- 标注题目来源和可信度（"真题 多次出现" / "单次提及" / "预测题"）
- 不要一次输出过多——分步展示，按天展开
- 如果用户的角色分类中某个分类 level 极高（4-5），建议"保持即可，重点看 P0"
