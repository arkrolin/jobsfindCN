# Mode 5: 算法练习 (algorithm-practice)

## 目标

交互式算法练习——出题、接收答案、评分、给出参考解法和改进建议，并更新用户画像中的算法 mastery。

## 前置条件

- `data/profile.yml` 应已存在
- 建议已有 `data/algorithm-progress.md`（计划）

## 执行流程

### Step 1: 选题

选择一道适合用户当前水平的题目：

**选题策略（按优先级）：**
1. 如果算法计划中有未完成的 P0 题 → 选 P0 题
2. 如果 P0 已完成 → 选 P1 题
3. 如果计划为空 → 从 `preferences.weak_areas_priority` 中最弱主题选题
4. 如果用户指定主题（"我想练动态规划"）→ 从该主题选题

**难度选择：**
- level 1-2 → Easy
- level 3 → Easy-Medium
- level 4 → Medium
- level 5 → Medium-Hard

**题目获取：**
使用 WebSearch 从 leetcode.cn 搜索目标主题+难度的题目。
如果用户有指定公司目标，优先搜索该公司的真题。

### Step 2: 出题

```
## 🧮 算法练习

**题目：**[题号]. [题名]
**难度：** [简单 / 中等 / 困难]
**主题：** [主题名称]
**来源：** [LeetCode / [公司名]真题]
**语言：** [用户偏好语言]

---

[题目描述（简洁版，含输入输出格式和示例）]

---

**请在下方编写代码。**
**时间限制：** [简单15min / 中等25min / 困难35min]
**要求：** 包含复杂度分析

准备好了告诉我，开始计时。
```

### Step 3: 接收答案

用户提交代码后：
1. 先快速检查代码逻辑是否正确
2. 检查是否有边界条件处理
3. 检查复杂度分析是否完备

### Step 4: 5 维评分

```
## 评分

**正确性：** [X]/5
- [正确的地方]
- [错误的地方（如有）]

**时间复杂度：** [X]/5
- 你的分析：[用户分析]
- 最优复杂度：[最优复杂度]
- 差距：[如有]

**空间复杂度：** [X]/5
- [同上]

**代码质量：** [X]/5
- 命名：[评价]
- 边界处理：[评价]
- 代码风格：[评价]

**思路表达：** [X]/5
- 清晰度：[评价]
- 是否解释了为什么这样解：[评价]

---

**总分：** [X.X]/5
```

### Step 5: 参考解法

```
## 参考解法

```[语言]
[最优解代码]
```

**解题思路：**
1. [步骤1]
2. [步骤2]
3. ...

**复杂度分析：**
- 时间：O(...)
- 空间：O(...)

**你的代码改进点：**
1. [具体改进——指向具体代码行]
2. [具体改进]
3. ...

**相关题目（拓展练习）：**
- [题号]. [题名] — [相同主题 / 变形]
- [题号]. [题名] — [进阶难度]
```

### Step 6: 更新画像

基于评分结果，更新 `data/profile.yml` 中的 `algorithm_mastery`：

1. 主题 `problems_done` + 1
2. `total_problems` + 1
3. 更新 `problems_by_difficulty`（easy/medium/hard 计数）
4. 更新 `last_practiced` 日期
5. 检查是否触发 **升级规则**：
   - 检查该主题最近 3 题（从 `algorithm-progress.md` 读取）是否都 ≥ 4.0/5
   - 是 → level + 1，提示用户
6. 检查是否触发 **降级规则**：
   - 检查该主题最近 3 题是否都 ≤ 2.0/5
   - 是 → level - 1（最低 1），温和提醒
7. 重新计算 `overall_level`
8. 更新 `weak_areas_priority`（如果 level 变化）
9. 更新 `last_leetcode_session` 日期

**画像更新提示：**
> 如果 level 升级： "🎉 恭喜！你的 [主题] 掌握度从 [旧] 升级到 [新]。继续保持！"
> 如果接近升级："你在这个主题上表现不错。再拿一道 ≥ 4.0/5 就能升级了。"
> 如果降级： "这一题有些难度。没关系，[主题]的基础部分建议再巩固一下。"

### Step 7: 记录进度

在 `data/algorithm-progress.md` 中追加本次练习记录：

```markdown
### [日期] — [题号].[题名]
- **主题：** [主题名称]
- **难度：** [简单/中等/困难]
- **得分：** [X.X]/5
- **用时：** [预估用时]
- **备注：** [一行记录，如"动态规划入门，边界条件处理需要加强"]
```

如果该题属于算法计划中的题目，标记为 `已完成 ✅`。

### Step 8: 下一步

> "要：
> 1. 再做一道 [同主题] 的题巩固
> 2. 换一个主题（当前薄弱项：[弱主题]）
> 3. 查看整体进展
>
> 你想怎么做？"

## 特殊情况处理

**用户做不出来：**
> "没关系，这道题对你目前有点挑战。要不要我：
> 1. 给一个提示（不减分）
> 2. 换一道更简单的同主题题
> 3. 直接看参考解法，然后你来解释思路"

**用户超时：**
> "时间到了。你现在可以：
> 1. 继续想（再给你 [X] 分钟）
> 2. 把目前的思路说出来，我给你反馈
> 3. 看解法后复述思路"

**连续刷题疲劳：**
如果用户连续 3 题后，主动提示：
> "已经连续刷了 3 题。刷题效率通常在 45 分钟后显著下降。要不要休息一下，或者换到八股文练习？"

## 注意事项

- 不要直接给答案——即使用户卡住，先给提示，再给思路，最后才给代码
- 评价时要具体——不能说"代码可以更好"，要指出第几行、怎么改
- 保持教练的语气——批评的是代码，不是人
- 如果用户使用非偏好语言答题，先接受答案，再提醒偏好
