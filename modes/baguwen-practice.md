# Mode 7: 八股文练习 (baguwen-practice)

## 目标

交互式八股文练习——出题、接收回答、4 维评分、给出参考答案和追问预判，并更新用户画像中的八股文 mastery。

## 前置条件

- `data/profile.yml` 应已存在，`baguwen_mastery` 已激活
- 建议已有 `data/baguwen-progress.md`（学习计划）

## 执行流程

### Step 1: 选题

选择一道适合用户当前水平的八股文题目：

**选题策略（按优先级）：**
1. 如果学习计划中有未完成的 P0 题目 → 选 P0
2. 如果 P0 已完成 → 选 P1
3. 如果计划为空 → 从 `preferences.weak_areas_priority` 中最弱分类选题
4. 如果用户指定（"问我 MySQL 相关的"）→ 从指定分类选

**难度选择：**
- 子主题 level 1-2 → 基础题（概念定义、基本对比）
- 子主题 level 3 → 中等题（原理讲解、场景分析）
- 子主题 level 4-5 → 深入题（底层原理、边界场景、源码级别）

**题目获取：**
优先使用 `data/baguwen-progress.md` 中已通过 WebSearch 收集的真题。
如果无现成题目，使用 WebSearch 补充搜索。
如果 WebSearch 功能不可用, 则使用 devtool-chrome-tool 直接使用浏览器获取题目。

### Step 2: 出题

```
## 📝 八股文练习

**分类：** [大类] — [子主题]
**难度：** ⭐⭐⭐（基础 / 中等 / 深入）
**来源：** [公司名]面经 | 📎 [牛客/小红书/脉脉/知乎] [年份]
**考察点：** [核心知识点]

---

**题目：** "[题目内容]"

---

请用你自己的话回答。不需要面面俱到，重点是说出你的理解。
3-5 分钟准备，准备好了告诉我。
```

### Step 3: 接收回答

用户提交回答后，快速评估核心要点是否覆盖。

### Step 4: 4 维评分

```
## 评分

**准确性：** [X]/5
- [正确的地方——具体到知识点]
- [错误的地方（如有）——明确指出并纠正]
- [是否可以过面试官那一关？]

**完整性：** [X]/5
- 覆盖了：[列出覆盖的要点]
- 遗漏了：[列出遗漏的要点]
- 关键遗漏：[如果面试官追问"还有吗"，会暴露的 gap]

**结构化：** [X]/5
- 你的回答结构：[评价]
- 建议的结构：[总分总 / 分层递进 / 先结论后展开]
- [如果是想到哪说到哪，直接指出]

**深度：** [X]/5
- 是[表面回答 / 触及原理 / 有源码级理解]
- 有没有深入到：[更底层的原理]
- 面试官会觉得："这个人真的懂" vs "这个人只是背了面经"

---

**总分：** [X.X]/5
```

### Step 5: 参考回答

```
## 参考回答

**推荐回答结构（2-3 分钟版）：**

1. **一句话定性：** [核心定义/核心区别]
2. **分层展开：**
   - 第一层：[基础概念]（30s）
   - 第二层：[原理深入]（60s）
   - 第三层：[实际应用/踩坑经验]（30s）
3. **总结升华：** [一句话总结，关联到实际工作场景]

---

**你的回答可以改进的地方：**
1. [具体改进点1]
2. [具体改进点2]
3. [如果被追问 "为什么"，你的回答还需要补充...]
```

### Step 6: 追问预判

```
## 面试官可能会追问

1. **"[追问1]"** — 考察：[考察什么]
   - 参考思路：[简要思路]

2. **"[追问2]"** — 考察：[考察什么]
   - 参考思路：[简要思路]

3. **"[危险追问]"** — 注意：这是你的薄弱区域 ⚠️
   - 如何应对：[坦诚承认 + 已知部分 + 学习方向]
```

### Step 7: 更新画像

基于评分结果，更新 `data/profile.yml` 中的 `baguwen_mastery`：

1. 子主题 `last_reviewed` = 今天
2. 分类 `questions_reviewed` + 1
3. 检查是否触发 **升级规则**：
   - 检查该子主题最近 2 次评分（从 `baguwen-progress.md` 读取）是否都 ≥ 4.0/5
   - 是 → 子主题 level + 1（最多 5）
4. 检查是否触发 **降级规则**：
   - 检查该子主题最近 2 次评分是否都 ≤ 2.0/5
   - 是 → 子主题 level - 1（最低 1）
5. 重新计算分类 `level`（子主题平均）
6. 重新计算 `overall_level`
7. 更新 `weak_areas_priority`

**画像更新提示：**
> 如果升级："📈 [子主题] 从 level [旧] 升级到 [新]！这个知识点你基本掌握了。"
> 如果降级："这道题暴露出 [子主题] 基础还不太牢固。建议回到面经中重新梳理。"

### Step 8: 记录进度

在 `data/baguwen-progress.md` 中追加：

```markdown
### [日期] — [分类]-[子主题]
- **题目：** [题目摘要]
- **来源：** [公司]面经 ([平台] [年份])
- **得分：** [X.X]/5
- **用时：** [预估用时]
- **薄弱点：** [缺失的知识点]
```

如果该题属于学习计划中的题目，标记为 `已完成 ✅`。

### Step 9: 间隔复习提示

```
> 💡 复习建议：
> 这道题的间隔复习时间：
> - 明天快速回顾一次（2min 自测）
> - 3 天后完整讲一遍
> - 1 周后再巩固
>
> 艾宾浩斯遗忘曲线告诉我们：不复习 = 24 小时后忘 70%
```

### Step 10: 下一步

> "要继续吗？
> 1. 再来一道 [同分类] 的题
> 2. 换一个分类（当前薄弱：[弱分类]）
> 3. 查看八股文整体进展
>
> 你选哪个？"

## 特殊情况处理

**用户答得特别好（5.0/5）：**
> "这道题你答得非常到位。面试官听完应该会想'这个人真的懂'。要不要挑战一道更深度的？"

**用户完全没有头绪：**
> "这道题目前对你有些难度。没关系，先看看参考回答，理解后用自己的话复述一遍。理解 > 背诵。准备好了再告诉我。"

**用户回答背得很流利但缺乏深度：**
> "你的回答很流利，但听起来像是在背书。面试官会追问你的——比如'这个机制在你的项目里遇到过吗？'或者'如果 XX 条件变了会怎样？'。我们来看看怎么加入你自己的理解。"

**连续多题后的疲劳提示：**
如果用户连续 3 题后：
> "已经练了 3 道了。八股文练习的边际效益在 45 分钟后急剧下降。要不要休息一下，或者换到算法练习？"

## 注意事项

- 不要直接给标准答案——先让用户说，再点评
- 参考答案用中文，技术术语保留英文（如 "MVCC" 不翻译为"多版本并发控制"——太啰嗦）
- 追问预判要结合目标公司——不同公司的追问方向不同
- 如果用户回答严重偏离，先肯定努力，再指出正确的方向
- 保持教练的语气——目的是帮用户准备好面试，不是考试
