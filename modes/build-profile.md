# Mode 2: 用户画像构建 (build-profile)

## 目标

构建完整的用户画像，包括算法能力自评、角色驱动的八股文能力自评，以及职业画像。画像写入 `data/profile.yml`。

## 前置条件

- `data/cv.md` 应已存在（如果不存在，先引导用户执行简历解析）
- 可以基于已有的 `data/profile.yml` 更新，也可以全新构建

## 执行流程

### Step 0: 加载现有数据

读取 `data/profile.yml`（如果存在）和 `data/cv.md`。如果 profile.yml 存在且完整，询问：
> "你的画像已存在（上次更新：[日期]）。要：
> 1. 完整重建画像
> 2. 仅更新特定部分（算法 / 八股文 / 职业信息）
>
> 哪个？"

### Step 1: 角色识别

**这是最关键的一步——决定了八股文的分类体系。**

根据简历和用户回答判断目标角色：
> "首先确认你的目标岗位：
> 1. Java 后端开发
> 2. Go 后端开发
> 3. C++ 开发
> 4. AI 算法工程师
> 5. Agent 开发工程师
> 6. 前端开发
> 7. 数据开发
> 8. 其他（请描述）"

如果用户选择了"其他"：
> "请简单描述你的目标岗位和主要技术栈。我会帮你匹配最接近的分类体系。"

确定角色后，从 `references/baguwen-role-taxonomy.md` 读取对应的分类体系，写入 `profile.yml` 的 `baguwen_mastery.role` 和 `baguwen_mastery.categories`。

### Step 2: 职业画像

基于简历和追问，填充：
> "再确认几个关键信息：
> - 你的目标公司有哪些？（可以按优先级列出）
> - 你的期望薪资范围？（年包，含年终奖和股票）
> - 有什么特别想去或不想去的公司吗？
> - 你觉得自己最突出的能力是什么？（2-3 个）"

更新 `profile.yml`：
- `target_roles.primary`
- `target_roles.companies`（tier1/tier2/tier3）
- `compensation`
- `narrative.superpowers`

### Step 3: 算法能力自评

> "现在评估你的算法水平。对于每个主题，告诉我：
> - 你做过多少题？
> - 能做到什么难度？（简单 / 中等 / 困难）"

按顺序评估 15 个算法主题：

| 主题 | 引导问题 |
|------|---------|
| 数组 | "数组/字符串相关题目你做得怎么样？（如：两数之和、合并区间）" |
| 动态规划 | "动态规划呢？（如：背包问题、最长子序列）" |
| 树/图 | "树和图相关的？（如：二叉树遍历、最短路径）" |
| 哈希表 | "哈希表？（如：字母异位词、两数之和变形）" |
| 排序/搜索 | "排序和搜索？（如：快速排序、二分查找变形）" |
| 贪心 | "贪心算法？（如：跳跃游戏、区间调度）" |
| 回溯 | "回溯算法？（如：全排列、N 皇后）" |
| 链表 | "链表操作？（如：反转链表、环形链表）" |
| 字符串 | "字符串处理？（如：KMP、字符串匹配）" |
| 栈/队列 | "栈和队列？（如：括号匹配、单调栈）" |
| 二分查找 | "二分查找？（如：搜索旋转数组、找峰值）" |
| 双指针 | "双指针？（如：三数之和、滑动窗口）" |
| 位运算 | "位运算？（如：位 1 的个数、异或操作）" |
| 数学 | "数学类题目？（如：素数、最大公约数）" |
| 设计 | "设计类题目？（如：LRU Cache、设计推特）" |

**不需要逐个详细问**——如果用户说"数组和链表我比较熟，动态规划和回溯基本没做过"，可以批量处理：熟的主题默认 level=3-4，不熟的主题默认 level=1-2，只对不确定的主题追问。

**level 映射：**
- "从没做过"/"完全不会" → level 1
- "做过几道简单题"/"了解概念" → level 2
- "能做简单题"/"偶尔做中等题" → level 3
- "能做中等题"/"偶尔做难题" → level 4
- "能做难题"/"竞赛水平" → level 5

写入 `profile.yml` 的 `algorithm_mastery.topics`。

### Step 4: 八股文能力自评

> "最后评估你的八股文水平。根据你的岗位角色（[角色名称]），我会按分类逐一确认。"

根据在 Step 1 激活的 `baguwen_mastery.categories`，逐个分类评估。每个分类询问：
> "你对 [分类名称] 的掌握如何？（1-5）这里面有特别薄弱或特别强的子主题吗？"

对每个子主题：
- 用户"很熟" / "面试没问题" → level 4
- 用户"了解" / "能答一部分" → level 3
- 用户"知道一点" / "看面经看过" → level 2
- 用户"完全不知道" / "没了解过" → level 1
- 只有用户能深入讲解原理 → level 5

写入 `profile.yml` 的 `baguwen_mastery.categories`。

### Step 5: 薄弱项识别

扫描整个画像，识别 level ≤ 2 的主题和子主题：
1. 算法主题中 level ≤ 2 的
2. 八股文子主题中 level ≤ 2 的
3. 按（目标公司出现频率）×（1 / 当前 level）排序
4. 写入 `preferences.weak_areas_priority`

### Step 6: 保存并总结

保存 `data/profile.yml`。

> "✅ 画像构建完成！
>
> **算法整体水平：**[X]/5 | 已做题数：[N]
> **八股文整体水平：**[X]/5 | 已复习题数：[N]
>
> **最需要加强的 3 个领域：**
> 1. [领域1] — 当前 [X]/5
> 2. [领域2] — 当前 [X]/5
> 3. [领域3] — 当前 [X]/5
>
> **建议下一步：**
> - 简历优化 → `/jobsfind-cn 简历优化`
> - 算法刷题计划 → `/jobsfind-cn 算法计划`
> - 八股文学习计划 → `/jobsfind-cn 八股文计划`
>
> 想从哪里开始？"

## 更新已有画像

如果用户只想更新部分内容：
- "更新算法水平" → 跳到 Step 3
- "更新八股文水平" → 跳到 Step 4
- "换目标岗位" → 跳到 Step 1（角色切换会提示重建八股文 mastery）
- "更新职业信息" → 跳到 Step 2

## 注意事项

- 尊重用户的自我评估——不要质疑"你真的能做中等题？"
- 如果用户明显高估或低估，用以下方式委婉确认：
  - "这个水平通常需要[XX]题量的积累，你觉得和你的情况匹配吗？"
- 角色切换时，保留旧的 mastery 数据作为历史记录（在 profile.yml 中标注）
