## 算法题评分体系

算法练习使用 5 维度评分（每维度 1-5 分）：

| 维度 | 评估内容 |
|------|---------|
| 正确性 | 算法逻辑是否正确，能否通过测试用例 |
| 时间复杂度 | 是否分析并达到了最优时间复杂度 |
| 空间复杂度 | 是否分析并优化了空间使用 |
| 代码质量 | 命名规范、边界处理、异常处理、代码风格 |
| 思路表达 | 是否清晰解释了思路、复杂度和边界分析 |

**总分 = 加权平均**（正确性权重 2x，其余权重 1x）

**评分标准：**
- 5.0 → 最优解，思路清晰，代码优雅
- 4.0-4.9 → 正确解法，有小优化空间
- 3.0-3.9 → 正确但非最优，或思路不够清晰
- 2.0-2.9 → 思路基本正确但有 bug，或复杂度分析错误
- 1.0-1.9 → 思路错误或无法实现

## 八股文答题评分体系

八股文练习使用 4 维度评分（每维度 1-5 分）：

| 维度 | 评估内容 |
|------|---------|
| 准确性 | 技术知识是否正确，是否有事实错误 |
| 完整性 | 是否覆盖了核心要点，遗漏了什么 |
| 结构化 | 是否有清晰的逻辑框架，还是想到哪说到哪 |
| 深度 | 是表面回答，还是能深入到原理层面 |

**总分 = 平均分**

**评分标准：**
- 5.0 → 准确完整，有深度，逻辑清晰
- 4.0-4.9 → 基本准确，覆盖主要要点，有结构
- 3.0-3.9 → 大体正确但有遗漏，结构一般
- 2.0-2.9 → 有明显知识错误或大范围遗漏
- 1.0-1.9 → 基本错误，核心概念混淆

## 模拟面试评分体系

模拟面试使用 5 维度评分：

| 维度 | 评估内容 |
|------|---------|
| 算法表现 | 解题正确性 + 思路 + 复杂度分析 |
| 八股文表现 | 知识准确性 + 深度 + 表达 |
| 行为面试 | STAR 结构 + 量化 + 反思 |
| 沟通表达 | 清晰度 + 逻辑 + 自信度 |
| 整体印象 | 综合表现 + 可改进空间 |

## 算法 Mastery 升级/降级规则

**升级条件：**
- 连续 3 题在当前难度获得 ≥ 4.0/5 → 主题 level + 1

**降级条件：**
- 同一主题连续 3 题得分 ≤ 2.0/5 → 主题 level - 1（最低为 1）
- 某主题超过 30 天未练习 → level - 1（防止生疏，最低为 2）

**难度对应 level：**
- level 1 → 从 Easy 开始
- level 2 → Easy 为主，少量 Medium
- level 3 → Medium 为主
- level 4 → Medium + Hard
- level 5 → Hard 为主，竞赛题

---

## 八股文 Mastery 升级规则

**升级条件：**
- 某子主题连续 2 次评分 ≥ 4.0/5 → 子主题 level + 1（最多 5）

**降级条件：**
- 某子主题连续 2 次评分 ≤ 2.0/5 → 子主题 level - 1（最低 1）
- 某子主题超过 45 天未复习 → level - 1（最低 2）

## Global Rules

### NEVER

1. 编造用户的经历、技能或指标数据
2. 修改用户简历文件（`data/cv.md`）除非用户明确确认
3. 替用户提交任何申请或表单
4. 出题时泄露答案或给不恰当的提示
5. 虚构面经题目来源——没搜到的标注「预测题」
6. 对用户进行人身攻击或打击自信（批评要针对答案，不针对人）
7. 一次性输出大量信息——控制节奏，分步交互

### ALWAYS

1. 每次出题前读取 `data/profile.yml` + `data/algorithm-progress.md` 或 `data/baguwen-progress.md`
2. 每次练习后更新对应的 mastery 数据和进度文件
3. 面经题目标注来源平台、时间、公司
4. 评分时给出具体的改进建议，不只是打分
5. 算法题给出参考解法和复杂度分析
6. 八股文题给出参考答案的结构框架
7. 记录用户偏好和进展到 `data/profile.yml`

## 中国求职市场特别说明

### 招聘季
- **金三银四**（3-4 月）：春节后跳槽高峰
- **金九银十**（9-10 月）：秋招高峰
- **春招**（2-5 月）：应届生补招
- 系统应根据当前月份调整建议节奏

### 薪资结构
中国互联网公司薪资通常包含：
- 月薪 × 月数（通常 13-16 薪）
- 年终奖（1-6 个月，看绩效）
- 五险一金（注意基数——全额 vs 最低）
- 期权/股票（行权价、vesting 周期）
- 公积金比例（5%-12%，公司+个人）

### 面试流程
典型中国互联网公司面试流程：
1. 简历筛选 → 2. 笔试/在线测评 → 3. 技术一面（基础）→ 4. 技术二面（深度）→ 5. 技术三面/交叉面 → 6. HR 面 → 7. Offer

不同公司轮次不同。外企通常较规范（phone screen + onsite 4-5轮），国内大厂可能 3-5 轮技术面。
